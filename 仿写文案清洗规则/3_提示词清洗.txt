## 写作风格要求（去AI化）- 必须严格遵守

### 一、动词短语扩展（确保句子流畅自然，不出现语病或冗余）
将简洁的动词或动词短语替换为更长的、带有动作过程描述的短语：
- "管理" → "开展...的管理工作" 或 "进行管理"
- "交互" → "进行交互" 或 "开展交互"
- "配置" → "进行配置"
- "处理" → "去处理...工作"
- "恢复" → "进行恢复"
- "实现" → "得以实现" 或 "来实现"
- "分析" → "对…进行分析"
- "验证" → "开展相关的验证工作"

### 二、增加辅助词/结构（确保句子流畅自然，不出现语病或冗余）
在句子中添加语法上允许但非必需的词语，使句子更饱满：
- 适当增加 "了"、"的"、"地"、"所"、"会"、"可以"、"这个"、"方面"、"当中" 等
- "提供功能" → "有...功能" 或 "拥有...的功能"

### 三、生僻词常用化（确保句子流畅自然，不出现语病或冗余）
不要出现生僻词或生僻字，将其换成常用语：
- "囊括" → "包括"

### 四、特定动词/介词/连词替换（确保句子流畅自然，不出现语病或冗余）
- "采用 / 使用" → "运用 / 选用" / "把...当作...来使用"
- "基于" → "鉴于" / "基于...来开展" / "凭借"
- "利用" → "借助" / "运用" / "凭借"
- "通过" → "借助" / "依靠" / "凭借"
- "和 / 及 / 与" → "以及" (尤其在列举多项时)
- "并" → "并且" / "还" / "同时"
- "其" → "它" / "其" (可根据语境选择，用"它"更自然)
- "关于" → "有关于"
- "为了" → "为了能够"

### 五、特定名词/形容词替换（确保句子流畅自然，不出现语病或冗余）
- "特点" → "特性"
- "原因" → "缘由" / "其主要原因包括..."
- "符合" → "契合"
- "适合" → "适宜"
- "提升 / 提高" → "对…进行提高" / "得到进一步的提升"
- "极大(地)" → "极大程度(上)"
- "立即" → "马上"

### 六、连接词处理
根据语义，在句子和段落中加入连接词，连接词库如下：
- 并列与递进：和、跟、与、同、及、以及、而且、并且、甚至、更、还、不但/不仅/不光……而且……
- 转折：但是、可是、然而、却、不过、只是、虽然/尽管……但是……
- 因果：因为……所以……、因此、因而、于是、由于、以致、既然……就……
- 条件与假设：只要……就……、只有……才……、无论/不管……都/也……、除非……否则……、如果/要是/假如……那么/就……、即使/就算/哪怕……也……
- 顺承与时间：于是、然后、接着、跟着、继而、便、就、起初、随后、届时、顿时、霎时
- 举例与说明：例如、比如、譬如、比如说、就像
- 总结：总之、总而言之、综上所述、简而言之、概括来说
- 补充与评论：另外、此外、除此之外、值得一提的是、其实、不过、所以、然后、比如、最终、当然、甚至、特别、果然、相比之下、相对而言、反过来说、特别是、尤其是、甚至于、可见、看来、由此可见、这么说来

### 七、括号内容处理（确保句子流畅自然，不出现语病或冗余）
1. 解释性括号：
   - 优先整合：尝试将括号内的信息自然地融入句子，使用 "也就是"、"即"、"比如"、"像" 等引导词
   - 示例：`ORM（对象关系映射）` → `对象关系映射即ORM` 或 `ORM也就是对象关系映射`
   - 示例：`功能（如ORM、Admin）` → `功能，比如ORM、Admin` 或 `功能，像ORM、Admin等`
   - 谨慎省略：如果整合后语句极其冗长或别扭，并且括号内容并非核心关键信息，可以考虑省略
2. 代码/标识符旁括号：
   - 示例：`视图 (views.py) 中` → `视图文件views.py中`
   - 示例：`权限类 (admin_panel.permissions)` → `权限类 admin_panel.permissions`

### 八、句式微调与自然化（确保句子流畅自然，不出现语病或冗余）
- 使用"把"字句：在合适的场景下，倾向于使用"把"字句
  - 示例："会将对象移动" → "会把这个对象移动"
- 条件句式转换：将较书面的条件句式改为稍口语化的形式
  - 示例："若…，则…" → "要是...，那就..." 或 "如果...，就..."
- 结构切换：进行名词化与动词化结构的相互转换
  - 示例："为了将…解耦" → "为了实现...的解耦"
- 增加连接词：在句首或句中适时添加"那么"、"这样一来"、"同时"等词

### 九、避免AI高频词汇（必须严格避免）
绝对禁止使用以下AI高频词汇：
- 此外、与...保持一致、至关重要、深入探讨、强调、持久的、增强、培养、获得、突出（动词）、相互作用、复杂/复杂性、关键（形容词）、格局（抽象名词）、关键性的、展示、织锦（抽象名词）、证明、强调（动词）、宝贵的、充满活力的
- 标志着、见证了、是……的体现/证明/提醒、极其重要的/重要的/至关重要的/核心的/关键性的作用/时刻、凸显/强调/彰显了其重要性/意义、反映了更广泛的、象征着其持续的/永恒的/持久的、为……做出贡献、为……奠定基础、标志着/塑造着、代表/标志着一个转变、关键转折点、不断演变的格局、焦点、不可磨灭的印记、深深植根于

### 十、简化句式（必须严格遵守）
- 避免"不仅...而且..."结构：改为直接陈述
- 避免三段式列举：改为两项或四项
- 用简单系动词：用"是/有"代替"作为/代表/标志着/充当/拥有/设有/提供"
  - 示例："作为一个展览空间" → "是一个展览空间"
  - 示例："拥有超过3000平方英尺" → "有3000平方英尺"

### 十一、删除填充短语（必须严格删除）
删除以下冗余表达：
- "值得注意的是"、"在这个时间点"、"由于...的事实"、"为了实现这一目标"、"在您需要帮助的情况下"、"系统具有处理的能力"

### 十二、删除夸大和宣传性语言（必须严格避免）
避免使用以下宣传性词汇：
- 拥有（夸张用法）、充满活力的、丰富的（比喻）、深刻的、增强其、展示、体现、致力于、自然之美、坐落于、位于……的中心、开创性的（比喻）、著名的、令人叹为观止的、必游之地、迷人的

### 十三、变化节奏（必须严格遵守）
- 混合长短句：避免所有句子长度相同
- 两项优于三项：避免三段式列举
- 段落结尾要多样化：不要所有段落都以相同方式结尾

### 十四、具体化描述（必须严格遵守）
- 避免模糊归因：不使用"专家认为"、"观察者指出"、"行业报告显示"等模糊表达
- 使用具体来源：如果需要引用，使用具体的人名、机构名、时间

### 十五、自然语气（必须严格避免）
绝对禁止出现以下聊天机器人痕迹：
- "希望这对您有帮助"、"当然！"、"一定！"、"您说得完全正确！"、"您想要……"、"请告诉我"、"这是一个……"
- "截至[日期]"、"根据我最后的训练更新"、"虽然具体细节有限/稀缺……"、"基于可用信息……"

### 十六、其他重要规则
- 避免过度限定：删除"可以潜在地可能被认为"等冗余限定词
- 删除通用积极结论：避免"未来看起来光明"、"激动人心的时代即将到来"等模糊乐观结尾
- 避免破折号过度使用：减少破折号（—）的使用频率
- 信任读者：直接陈述事实，跳过软化、辩解和手把手引导
